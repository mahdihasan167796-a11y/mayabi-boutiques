# Mayabi Boutiques — Next.js প্রজেক্ট (ফাইনাল ভার্সন)

## ✅ এই ভার্সনে যা যা আছে
1. **ডায়নামিক প্রোডাক্টস** — প্রোডাক্ট এখন Supabase ডাটাবেজ থেকে লোড হয় (`lib/products.ts`)
2. **অ্যাডমিন প্যানেল (`/admin`)** — পাসওয়ার্ড-প্রোটেক্টেড, প্রোডাক্ট যোগ/ডিলিট ও অর্ডার স্ট্যাটাস ম্যানেজমেন্ট
3. **অর্ডার ইমেইল নোটিফিকেশন** — নতুন অর্ডার এলে Resend দিয়ে আপনার ইমেইলে অ্যালার্ট
4. **বিকাশ/নগদ ম্যানুয়াল পেমেন্ট** — ক্যাশ অন ডেলিভারির পাশাপাশি বিকাশ/নগদ + ট্রানজেকশন আইডি অপশন

---

## 📁 নতুন/পরিবর্তিত ফাইল স্ট্রাকচার
```
mayabi-boutiques/
├── middleware.ts                        → /admin ও /api/admin রুট প্রোটেক্ট করে
├── app/
│   ├── admin/
│   │   ├── login/page.tsx               → অ্যাডমিন লগইন পেজ
│   │   ├── page.tsx                     → অ্যাডমিন ড্যাশবোর্ড (সার্ভার — ডাটা লোড করে)
│   │   └── admin-dashboard.tsx          → প্রোডাক্ট/অর্ডার ট্যাব (ক্লায়েন্ট)
│   ├── api/
│   │   ├── orders/route.ts              → পাবলিক অর্ডার সাবমিট + ইমেইল নোটিফিকেশন
│   │   └── admin/
│   │       ├── login/route.ts, logout/route.ts
│   │       ├── products/route.ts        → GET (লিস্ট) / POST (ছবি আপলোডসহ নতুন প্রোডাক্ট)
│   │       ├── products/[id]/route.ts   → DELETE
│   │       ├── orders/route.ts          → GET (সব অর্ডার)
│   │       └── orders/[id]/route.ts     → PATCH (স্ট্যাটাস আপডেট)
│   └── product/[slug]/product-detail-client.tsx  → এখন বিকাশ/নগদ/COD পেমেন্ট অপশনসহ
├── lib/
│   ├── categories.ts       → স্ট্যাটিক ক্যাটাগরি তালিকা (নাম/ট্যাগ/ছবি)
│   ├── products.ts         → Supabase থেকে প্রোডাক্ট queries (getProductsByCategory, getProductBySlug...)
│   ├── supabase.ts         → সার্ভার-সাইড Supabase ক্লায়েন্ট
│   ├── email.ts            → Resend দিয়ে অর্ডার নোটিফিকেশন
│   ├── admin-auth.ts       → পাসওয়ার্ড সেশন হ্যাশিং হেল্পার
│   └── slugify.ts          → নতুন প্রোডাক্টের জন্য স্লাগ জেনারেটর
└── supabase/schema.sql     → products + orders টেবিল তৈরির SQL
```

## 🗄️ ধাপ ১ — Supabase সেটআপ (ডাটাবেজ)
1. https://supabase.com -এ ফ্রি অ্যাকাউন্ট খুলে একটি প্রজেক্ট বানান।
2. **SQL Editor**-এ গিয়ে `supabase/schema.sql`-এর পুরো কনটেন্ট পেস্ট করে **Run** চাপুন — এতে `products` ও `orders` দুটো টেবিল তৈরি হবে।
3. **Storage** মেনুতে গিয়ে **"New bucket"** চাপুন, নাম দিন ঠিক এভাবে: `product-images`, এবং **Public bucket** টগল অন করুন (নাহলে ছবি ব্রাউজারে দেখা যাবে না)।
4. **Project Settings → API**-তে গিয়ে `Project URL` ও `service_role` (secret) কী কপি করুন।

## 🔑 ধাপ ২ — এনভায়রনমেন্ট ভ্যারিয়েবল
1. `.env.local.example` ফাইলের নাম বদলে `.env.local` করুন।
2. প্রতিটা মান বসান — নিচে প্রতিটির উৎস দেওয়া হলো:

| ভ্যারিয়েবল | কোথা থেকে পাবেন |
|---|---|
| `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` | Supabase → Project Settings → API |
| `ADMIN_PASSWORD` | নিজে একটি শক্তিশালী পাসওয়ার্ড ঠিক করুন |
| `RESEND_API_KEY` | https://resend.com ফ্রি অ্যাকাউন্ট → API Keys |
| `ADMIN_NOTIFY_EMAIL` | যে ইমেইলে অর্ডার এলার্ট পেতে চান (আপনার নিজের ইমেইল) |
| `ORDER_FROM_EMAIL` | ডিফল্ট `onboarding@resend.dev` রাখুন (কাস্টম ডোমেইন ভেরিফাই না করা পর্যন্ত) |
| `NEXT_PUBLIC_BKASH_NUMBER`, `NEXT_PUBLIC_NAGAD_NUMBER` | আপনার বিকাশ/নগদ পার্সোনাল বা মার্চেন্ট নাম্বার |

⚠️ **`SUPABASE_SERVICE_ROLE_KEY` কখনো পাবলিক করবেন না** — এটা শুধু সার্ভার-সাইড কোডে (`.env.local` / Vercel Environment Variables) থাকবে, কখনো ব্রাউজারে যাবে না।

## ▶️ ধাপ ৩ — লোকালি রান করুন
```bash
npm install
npm run dev
```
http://localhost:3000 এ ব্রাউজারে দেখুন।

## 🔐 ধাপ ৪ — অ্যাডমিন প্যানেল ব্যবহার
1. যান: `http://localhost:3000/admin` → এটি স্বয়ংক্রিয়ভাবে লগইন পেজে নিয়ে যাবে।
2. `.env.local`-এ দেওয়া `ADMIN_PASSWORD` দিয়ে লগইন করুন।
3. **"প্রোডাক্ট সমূহ"** ট্যাবে গিয়ে নাম, ক্যাটাগরি, দাম ও ছবি দিয়ে নতুন প্রোডাক্ট যোগ করুন — সাথে সাথে সাইটে লাইভ হয়ে যাবে।
4. **"অর্ডার সমূহ"** ট্যাবে সব কাস্টমার অর্ডার দেখুন এবং ডানপাশের ড্রপডাউন থেকে স্ট্যাটাস (Pending → Confirmed → Shipped → Delivered) আপডেট করুন।

## 📧 ধাপ ৫ — অর্ডার নোটিফিকেশন টেস্ট
কোনো প্রোডাক্ট পেজ থেকে একটি টেস্ট অর্ডার সাবমিট করুন। কয়েক সেকেন্ডের মধ্যে `ADMIN_NOTIFY_EMAIL`-এ একটি ইমেইল আসবে। না এলে টার্মিনালের লগ চেক করুন — সেখানে এরর মেসেজ দেখাবে।
> Resend-এর ফ্রি/স্যান্ডবক্স মোডে শুধু আপনার নিজের রেজিস্টার্ড ইমেইলেই মেইল যাবে। কাস্টম ডোমেইন ভেরিফাই করলে যেকোনো ইমেইলে পাঠানো যাবে (Resend ড্যাশবোর্ডে Domains সেকশনে গাইড আছে)।

## 💳 ধাপ ৬ — বিকাশ/নগদ পেমেন্ট
`.env.local`-এ দেওয়া `NEXT_PUBLIC_BKASH_NUMBER` / `NEXT_PUBLIC_NAGAD_NUMBER` নাম্বারটাই কাস্টমারকে দেখানো হবে সেন্ড মানি করার জন্য। কাস্টমার ট্রানজেকশন আইডি দিলে সেটা অর্ডারের সাথে সেভ হয়ে অ্যাডমিন প্যানেলে ও নোটিফিকেশন ইমেইলে দেখা যাবে — আপনি ম্যানুয়ালি বিকাশ/নগদ অ্যাপে গিয়ে সেই টাকা রিসিভ হয়েছে কিনা মিলিয়ে নেবেন।

## ⚙️ ধাপ ৪.৫ — ফুটার সোশ্যাল লিংক ও ফোন নাম্বার (Settings)
অ্যাডমিন প্যানেলের **"সেটিংস"** ট্যাবে গিয়ে Facebook, Instagram, TikTok, Messenger লিংক এবং যোগাযোগের ফোন নাম্বার বসান — সাথে সাথে হোমপেজের ফুটারে দেখাবে, কোনো কোড ছোঁয়া লাগবে না। কোনো ফিল্ড খালি রাখলে সেই আইকনটি ফুটারে দেখানো হবে না।

## ✅ এই ভার্সনে আরও যা ঠিক করা হয়েছে
- হোমপেজের মূল বাটনগুলো (কালেকশন দেখুন, এখনি অর্ডার করুন, হেডারের "অর্ডার করুন", কম্বো প্যাকেজের বাটনসমূহ) এখন সব **ফিচারড কালেকশন** সেকশনে নিয়ে যায়।
- ক্যাটাগরি চিপ-মেনুর নিচের স্ক্রলবার সম্পূর্ণ লুকানো হয়েছে (মোবাইলে সোয়াইপ করে স্ক্রল করা যাবে, কিন্তু কোনো সাদা দাগ/স্ক্রলবার দেখাবে না)।
- মোবাইলে হ্যামবার্গার মেনু যোগ করা হয়েছে (আগে মোবাইলে নেভিগেশন লিংকগুলো দেখাই যেত না)।
- হেডার, হিরো, চেকআউট ফর্ম, পেমেন্ট মেথড সিলেক্টর — সবকিছু ছোট স্ক্রিনে (৩২০px থেকে) ভেঙে না গিয়ে সুন্দরভাবে সাজে।

## 🚀 ধাপ ৭ — Vercel-এ লাইভ করা
1. GitHub-এ প্রজেক্টটি পুশ করুন (`.env.local` অটোমেটিক বাদ যাবে, `.gitignore`-এ আছে)।
2. Vercel-এ ইমপোর্ট করুন।
3. Vercel ড্যাশবোর্ড → **Project → Settings → Environment Variables**-এ উপরের সবগুলো ভ্যারিয়েবল যোগ করুন।
4. Deploy করুন। এরপর `https://yourdomain.com/admin` থেকে অ্যাডমিন প্যানেল অ্যাক্সেস করতে পারবেন।

## ⚠️ মনে রাখবেন
- প্রথমবার সাইট লাইভ করার পর ডাটাবেজ খালি থাকবে — `/admin` থেকে প্রোডাক্ট যোগ না করা পর্যন্ত ক্যাটাগরি পেজগুলোতে "কোনো প্রোডাক্ট নেই" দেখাবে (এটাই স্বাভাবিক, ইচ্ছাকৃতভাবে রাখা হয়েছে)।
- ভবিষ্যতে চাইলে প্রোডাক্টে একাধিক কালার-ভ্যারিয়েন্ট/একাধিক ছবি ম্যানেজ করার UI, বা অ্যাডমিন প্যানেলে সার্চ/পেজিনেশন যোগ করে দিতে পারি — বললেই করে দেব।
- `ADMIN_PASSWORD` পরিবর্তন করলে আগের লগইন সেশন আর কাজ করবে না (নতুন করে লগইন করতে হবে) — এটা স্বাভাবিক আচরণ।
